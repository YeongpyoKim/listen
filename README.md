# 동네 한 바퀴, 은혜 한 바구니 🧺

경청 프로젝트 2기 · 작은 실천 — 우리 동네 21개 가게를 **인스타그램형 감성 갤러리**와 **상점별 스토리 페이지**로 소개하고, 보조 수단으로 **로컬 챗봇**(동네 길잡이)을 연결한 POC입니다.

> ⚠️ **이 프로젝트는 로컬 전용 POC입니다.** 외부 무료 호스팅에 절대 올리지 않습니다.
> 다만 구조는 실제 배포에도 그대로 쓸 수 있도록 정적 사이트 + JSON 데이터로 설계했습니다.

---

## 빠르게 실행하기

```powershell
# 1) 데이터/이미지 빌드 (엑셀 + 사진 폴더 → site/data/stores.json, site/assets/*)
python build_site.py

# 2) 로컬 서버 실행 (http://127.0.0.1:8000)
python serve.py
```

브라우저가 자동으로 열립니다. (수동: http://127.0.0.1:8000/index.html )

> `file://` 로 직접 열면 `fetch`(stores.json)가 막히므로 **반드시 `serve.py`(로컬 서버)로** 여세요.

---

## 폴더 구조

```
Listen/
├─ 상점 리스트.xlsx        # 원본 데이터 (사실 정보 + 네이버 링크)
├─ <상점명>/               # 상점별 사진 폴더 (가장 앞번호 = 대표사진)
├─ site_content.py         # 감성 콘텐츠(메시지/스토리/주일안내/주소/전화) — 사람이 큐레이션
├─ build_site.py           # 빌드 스크립트 (엑셀 + 폴더 + 콘텐츠 → JSON/이미지)
├─ serve.py                # 로컬 전용 미리보기 서버
└─ site/                   # ← 산출물 (이 폴더만 있으면 어디서든 정적 서빙 가능)
   ├─ index.html           # 인스타형 메인 그리드
   ├─ store.html           # 상점 상세(감성 서브 페이지, ?id=s01)
   ├─ css/styles.css
   ├─ js/app.js            # 메인 그리드 렌더링/검색/필터
   ├─ js/store.js          # 상세 페이지 렌더링
   ├─ js/chatbot.js        # 동네 길잡이 챗봇 (로컬/오프라인)
   ├─ data/stores.json     # 빌드 산출 데이터
   └─ assets/<id>/         # 빌드가 복사한 사진
```

## 아키텍처 결정 (왜 이렇게 했나)

- **메인은 사진과 웹사이트**, 챗봇은 보조 — 정적 HTML/CSS/JS + `stores.json` 단일 데이터로 구성했습니다.
  - 빌드만 다시 돌리면 데이터가 갱신되고, `site/` 폴더 전체가 자기완결적이라 **실제 배포(정적 호스팅)에도 그대로** 올릴 수 있습니다.
- **챗봇**은 `stores.json` 기반 **규칙형 검색 봇**으로, 키 없이 완전 오프라인 동작합니다(POC·로컬 요구사항 충족).
  - 추후 실제 LLM 이 필요하면 `site/js/chatbot.js` 의 `endpoint` 에 **로컬 백엔드 URL**(예: FastAPI + RAG)만 지정하면 그쪽으로 질의를 보냅니다. 프런트는 그대로 재사용됩니다.
- **Streamlit 대신 정적 사이트**를 택한 이유: 사진 중심의 인스타형 그리드/서브페이지 같은 **정밀한 감성 UI**는 정적 사이트가 훨씬 자유롭고 가볍습니다. (Streamlit 은 데이터 앱엔 강하지만 픽셀 단위 감성 레이아웃엔 제약이 큽니다.)

## 대상 독자 = 주일에 이 동네를 찾는 성도

- 각 카드/상세에 **주일 영업 여부 배지**를 명확히 표시합니다.
  - `주일 영업` / `주일 격주 휴무(⚠️ 빵굼터)` / `주일 확인 필요(📞)`
- 정보가 불확실한 곳은 임의로 단정하지 않고 **"방문 전 전화 확인"** 으로 안내해 헛걸음을 막습니다.

## 사진이 없는 가게 (블리스냅 · 태문네 · 은행나무집 · 김밥365)

- 사진 폴더가 없어, 저작권을 침해하지 않도록 **감성 그라디언트 카드**로 대표 이미지를 대신했습니다.
- 정보(주소/전화/영업/스토리)는 네이버 지도·리뷰를 참고해 채웠습니다.
- 나중에 해당 **상점명 폴더에 실제 사진을 넣고 `python build_site.py` 만 다시 실행**하면 자동으로 대표사진/갤러리가 채워집니다.

## 내용 수정 방법

- **사실 정보**(메뉴/가격/시간/좌석/주차): `상점 리스트.xlsx` 수정 후 `python build_site.py`
- **감성 문구·스토리·주일 안내·주소/전화**: `site_content.py` 수정 후 `python build_site.py`
- **사진**: 상점 폴더에 추가/교체 후 `python build_site.py` (가장 앞번호가 대표)
